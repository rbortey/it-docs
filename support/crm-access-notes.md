# CRM Access Notes

CRM access is handled through named accounts and MFA. Shared credentials should not be used for normal operations.

## Common Issues

- User forgot password.
- MFA prompt fatigue or repeated prompts.
- Browser session expired during customer update.
- Vendor asks for temporary confirmation during support window.

## Support Handling

- Verify the reporter identity.
- Confirm whether MFA is expected.
- Ask the user not to approve prompts they did not initiate.
- Escalate repeated prompts if they continue after password reset.

## Watch Items

Public staff profiles and old repository notes can make CRM usernames easier to guess. Treat unexpected CRM login attempts as a security signal, even when MFA prevents access.
